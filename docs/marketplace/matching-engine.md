# Smart Matching Engine Specifications

The **Matching Engine** represents the core routing logic mapping microtask guidelines to optimal human intelligence nodes.

## Evaluated Factors

1. **Contributor Skills**
   Looks at direct overlaps between a contributor's declared skill set and the task category/tags. Full match awards high scores, while missing skills slightly penalty compatibility.

2. **Trust Score and Accuracy Tiers**
   Ensures contributors meet minimum security and validation rates. Excellent track records boost scores with multipliers.

3. **Language Fluency**
   Analyzes ISO language codes (e.g. `es-ES`, `en-US`) to match contributors to localized datasets. Non-proficient candidates are filtered out to prevent low-integrity data entry.

4. **Geographical Constraints**
   Restricts high-confidentiality campaigns to particular sovereign boundaries (e.g. US only).

5. **Device Capabilities**
   Checks task requirements (e.g., GPU resources, high memory, microphone capture, image validation) against the user’s active profile capabilities.

6. **Reward and Difficulty Preferences**
   Tackles churn and contributor fatigue by recommending tasks that align with stated difficulty preferences and preferred reward tiers (Coins, XP, or Mixed).

## Calculated Outputs

- **Compatibility Score (0-100)%**: Overall alignment index.
- **Expected Success Rate (0-100)%**: Predicted precision of contributor's consensus feedback.
- **Expected Completion Time**: Custom projection factoring in the contributor’s level and speed multipliers.
- **Estimated Earnings**: Final coin yields incorporating profile leveling multipliers.
- **Recommendation Confidence**: Ranked categorization (`Low`, `Medium`, `High`) describing match reliability.
